Initial VPS Configuration & K3s Installation

Pre-configuration Essentials

Before installing K3s, both VPS instances require essential pre-configuration including unique hostnames, passwordless SSH, system updates, and firewall configuration.

Server Node (Ubuntu 24.04)

sudo hostnamectl set-hostname k3s-server
echo "127.0.1.1 k3s-server" | sudo tee -a /etc/hosts

sudo apt update && sudo apt upgrade -y
sudo apt install -y curl

sudo swapoff -a
sudo sed -i '/swap/s/^/#/' /etc/fstab

sudo ufw allow 6443/tcp
sudo ufw allow 22/tcp
sudo ufw enable

Agent Node (Rocky 9)

sudo hostnamectl set-hostname k3s-agent
echo "127.0.1.1 k3s-agent" | sudo tee -a /etc/hosts

sudo dnf update -y
sudo dnf install -y curl

sudo swapoff -a
sudo sed -i '/swap/s/^/#/' /etc/fstab

sudo firewall-cmd --permanent --add-port=6443/tcp
sudo firewall-cmd --permanent --add-service=ssh
sudo firewall-cmd --reload

Rocky Linux 9 Specific Consideration

On RHEL-based systems like Rocky Linux 9, you may encounter cgroup issues. [172] Add kernel parameters before installation:

sudo grubby --args="cgroup_memory=1 cgroup_enable=memory" --update-kernel="$(grubby --default-kernel)"
sudo reboot

K3s Server Installation

Install K3s on the Ubuntu server node, retrieve the node token, and configure kubectl access.

# Install K3s server
curl -sfL https://get.k3s.io | sh -

# Verify service status
sudo systemctl status k3s

# Retrieve node token
sudo cat /var/lib/rancher/k3s/server/node-token

# Configure kubectl
mkdir -p ~/.kube
sudo cp /etc/rancher/k3s/k3s.yaml ~/.kube/config
sudo chown $USER:$USER ~/.kube/config

Agent Node Joining

Join the Rocky Linux agent node to the cluster using the retrieved node token.

# Join agent to cluster
curl -sfL https://get.k3s.io | \
  K3S_URL=https://<SERVER_IP>:6443 \
  K3S_TOKEN=<NODE_TOKEN> sh -

# Verify cluster status
kubectl get nodes
Both nodes should show status "Ready" in the cluster

MetalLB and Ingress Controllers

Installation Order: Why MetalLB First

MetalLB must be installed before ingress controllers to provide external IP addresses for LoadBalancer services. [94]

1

MetalLB Installation

Provides LoadBalancer IP allocation infrastructure

# Enable strictARP for kube-proxy
kubectl get configmap kube-proxy -n kube-system -o yaml | \
  sed -e "s/strictARP: false/strictARP: true/" | \
  kubectl apply -f - -n kube-system

# Install MetalLB
kubectl apply -f https://raw.githubusercontent.com/metallb/metallb/main/config/manifests/metallb-native.yaml

# Verify installation
kubectl get pods -n metallb-system
2

Configure IP Address Pool

Define available IP range for LoadBalancer services [114] 

# metallb-config.yaml
apiVersion: metallb.io/v1beta1
kind: IPAddressPool
metadata:
  name: first-pool
  namespace: metallb-system
spec:
  addresses:
  - 192.0.2.100-192.0.2.110

---
apiVersion: metallb.io/v1beta1
kind: L2Advertisement
metadata:
  name: example
  namespace: metallb-system
spec:
  ipAddressPools:
  - first-pool

# Apply configuration
kubectl apply -f metallb-config.yaml

Nginx Ingress Controller

Recommended for most use cases with extensive feature support.

# Add Helm repository
helm repo add ingress-nginx https://kubernetes.github.io/ingress-nginx
helm repo update

# Install Nginx Ingress
helm install nginx-ingress ingress-nginx/ingress-nginx \
  --namespace ingress-nginx \
  --create-namespace

# Configure for MetalLB
kubectl patch svc ingress-nginx-controller \
  -n ingress-nginx \
  -p '{"spec":{"type":"LoadBalancer"}}'

HAProxy Ingress Controller

Alternative for high-performance requirements and advanced load balancing.

# Add Helm repository
helm repo add haproxytech https://haproxytech.github.io/helm-charts
helm repo update

# Install HAProxy Ingress
helm upgrade -i haproxy haproxytech/kubernetes-ingress \
  --create-namespace \
  --namespace haproxy-ingress

# Configure for MetalLB
kubectl patch svc haproxy-kubernetes-ingress \
  -n haproxy-ingress \
  -p '{"spec":{"type":"LoadBalancer"}}'

Longhorn Distributed Storage

Installation & Prerequisites

Longhorn provides enterprise-grade persistent storage with replication and backup capabilities. [118]

Install Prerequisites on All Nodes

Ubuntu 24.04:

sudo apt update
sudo apt install -y open-iscsi nfs-common
sudo systemctl enable iscsid
sudo systemctl start iscsid

Rocky 9:

sudo dnf install -y iscsi-initiator-utils nfs-utils
sudo systemctl enable iscsid
sudo systemctl start iscsid
# Install Longhorn
kubectl apply -f https://raw.githubusercontent.com/longhorn/longhorn/v1.8.1/deploy/longhorn.yaml

# Verify installation
kubectl get pods -n longhorn-system
kubectl get crds | grep longhorn

Alternative: Helm Installation

For more control over configuration parameters: [120]

Configuration & Management

Access Longhorn UI

# Port-forward to access UI
kubectl port-forward -n longhorn-system \
  svc/longhorn-frontend 8080:80

# Access at http://localhost:8080

Set as Default Storage

# Unset local-path as default
kubectl patch storageclass local-path -p \
  '{"metadata": {"annotations":{"storageclass.kubernetes.io/is-default-class":"false"}}}'

# Set Longhorn as default
kubectl patch storageclass longhorn -p \
  '{"metadata": {"annotations":{"storageclass.kubernetes.io/is-default-class":"true"}}}'

Persistent Volume Example

Create a PVC for database applications: [126] 

apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: db-pvc
spec:
  accessModes:
    - ReadWriteOnce
  storageClassName: longhorn
  resources:
    requests:
      storage: 10Gi

Production Application Deployment

Rancher Cluster Management

Deploy Rancher for comprehensive Kubernetes cluster management with web UI and multi-cluster support.

1. Install cert-manager

helm repo add jetstack https://charts.jetstack.io
helm repo update

helm install cert-manager jetstack/cert-manager \
  --namespace cert-manager \
  --create-namespace \
  --version v1.14.2 \
  --set installCRDs=true

2. Deploy Rancher

helm repo add rancher-latest https://releases.rancher.com/server-charts/latest
kubectl create namespace cattle-system

helm install rancher rancher-latest/rancher \
  --namespace cattle-system \
  --set hostname=rancher.barrzen.com \
  --set bootstrapPassword=admin \
  --set ingress.tls.source=letsEncrypt

PostgreSQL Database

Stateful database deployment with persistent storage and high availability.

# Create secret for PostgreSQL password

# Deploy StatefulSet with Longhorn storage
kubectl apply -f postgres-statefulset.yaml
